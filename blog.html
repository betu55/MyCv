Does Relative Size matter?

Well it is known that the browsers we use now-a-days have more features and tools within them.

But in the early ages all of the web pages that were being developed were perfected using pixele values

since there were no fancy features going on in almost all browsers. In this age tho most browsers have a lot

of cool features, and one that almos everyone knows is the ZOOM feature. So instead of using pixeles to

size elements in css it is more relevant to use relative sizing like: %, em, rem instead of pixel values.

Users must be able to zoom the viewport without loss of content and functionality, or restrictions imposed by

CSS values or viewport scaling settings. They must be able to resize text without assistive technology up to 200%.

It is easy to make our website user friendly by using relative size to change the size of the elements in css.

BY now I guess the answer for the question above is clear.

CSS3 Features

CSS3 version introduced new features that allowed developers to style HTML elements with less CSS code. At first

browsers did not support CSS3 features, and it took a while for them to become fully compatable with CSS3. Oh and

by the way CSS3 is backwards-compatible with former CSS versions. It means that css3 features work on web pages

using older CSS. The following are overviews of new features in CSS3.


Box Shadow: Adds a shadow to an element.

Opacity: Makes elements see-through or completely transparent.

Attribute Selectors: instead of ID and Class selectors developers can now select elements via their attributes.

New Colors:new colores are added in CSS3, for example RGBA, HSL, HSLA and Gradient Colors
